# Monica 项目合规检查标准流程 v1.0

_建立时间：2026-03-27_
_适用场景：任何对外发布（网站/GitHub/文档/公开内容）_

---

## 为什么要合规检查

Monica 知道 Shirley 太多隐私信息（日记、情感、财务、健康等）。
这些信息在内部记录没问题，但如果泄露到外部，会造成实际伤害。

合规检查是上线前的最后一道防线，不是可选项。

---

## 三阶段合规检查

### Stage 1：隐私检查（上线前必须）

**检查范围：**
- 所有文字内容（正文、标题、描述、注释）
- 所有图片/附件（截图里的文字也可能泄露）
- 所有链接（链接文字也可能暴露信息）

**隐私扫描清单：**

| 类别 | 关键词 | 处理方式 |
|------|--------|---------|
| 财务数据 | 50万、储蓄、工资、收入、本金、亏损 | 删除或泛化为"有储蓄目标"/"在投资" |
| 情感关系 | 分手、现任、前任、男友、离婚、情感 | 泛化为"感情经历" |
| 健康/去世 | 去世、死亡、抑郁、失眠、病、手术 | 删除，泛化为"关注健康" |
| 日记内容 | 日记、十年、记录、读后感 | 删除，用"深度阅读"泛化 |
| 具体行程 | 清明、河南、自驾、4天3晚 | 泛化为"一次短途旅行" |
| 健康状况 | 胃、身体、累不累 | 删除，不主动提 |
| 证件/密码 | 身份证、护照、银行卡、密码 | 删除 |
| 家人信息 | 爸妈、弟弟、去世亲人名字 | 泛化为"家人" |
| 私密对话 | 引用的聊天记录、截图内容 | 泛化或删除 |

**扫描顺序：**
1. 读完整内容（不跳步）
2. 凭记忆扫关键词（上面清单）
3. 每发现一个 → 立即修改
4. 改完再发布

---

### Stage 2：准确性检查

**检查清单：**
- [ ] 数字是否用工具验证过？（日期/天数/百分比/金额）
- [ ] 截图/图片是否用识图工具确认过内容？
- [ ] 有没有"推论当事实"的问题？
- [ ] 链接是否都是有效的？
- [ ] 外部引用（GitHub/Gitee等）是否可访问？

---

### Stage 3：一致性检查

**检查清单：**
- [ ] 内容是否符合 SOUL.md 的语气和价值观？
- [ ] 是否有违背 Red Lines 的内容？（不外传私密信息）
- [ ] 对外身份是否一致？（职业描述、工作项目是否合规）
- [ ] Monica 的定位是否准确？（数字分身，不是普通助理）

---

## 检查后汇报格式

```
【合规检查】项目名称

✅ 隐私检查：通过（已泛化 X 处）
  - [具体位置] 50万 → "人生目标"
  - [具体位置] 日记内容 → 已删除

✅ 准确性检查：通过
  - 数字已工具验证
  - 链接全部有效

✅ 一致性检查：通过
  - 语气符合 SOUL.md
  - 无私密信息外泄

【结论】可以上线 / 需要修复后上线
```

## 快速检查（5分钟内）

如果项目很简单（改了一句话/加了一个链接），用快速版：

**必检项：**
1. 这一条内容有没有 L1 隐私？（财务/日记/去世/证件）
2. 截图里有没有敏感信息？
3. 链接指向的内容有没有隐私问题？

---

## 示例：Monica Magazine 反思文章

**发布前合规检查流程：**

1. **隐私扫描**
   - "50万年储蓄目标" → 泛化为"人生目标" ✅
   - "小天开车" → 删除具体同行人信息 ✅
   - "十年日记分析" → 泛化为"深度阅读" ✅
   - "家人在XX年去世" → 删除具体信息 ✅

2. **准确性扫描**
   - 日期是否正确？（用工具验证天数计算）
   - 外部链接是否有效？

3. **一致性扫描**
   - 语气是否 Monica 风格？
   - 是否符合"数字分身"的定位？

## 触发场景

**必须执行完整合规检查：**
- 首次发布网站/GitHub 仓库
- 首次发布新的内容类型（文章/反思/作品集）
- 修改涉及新页面或新文章

**可执行快速检查：**
- 只改了一句话
- 只更新了一个链接
- 只改了一个图片

**任何时候：发现疑似隐私内容 → 先检查再发布。**
